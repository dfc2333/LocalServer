# LocalServer 群聊功能测试说明

唐棠已经在TT分支添加了群聊功能，以下是测试步骤：

## 1. 启动服务器
确保已安装Flask依赖，然后运行：
```bash
cd LocalServer
python Server.pyw
```

服务器将在 `http://0.0.0.0:1145` 启动。

## 2. 测试前提条件
需要先有用户登录。可以通过以下方式：
1. 访问 `http://localhost:1145/login` 设置用户名
2. 或者在 `userlist.txt` 中添加用户IP映射

## 3. 测试群聊API

### 3.1 创建群聊
```
GET http://localhost:1145/group/create?name=测试群聊
```
响应示例：
```json
{"success": true, "group_id": "1708901234567", "group_name": "测试群聊"}
```

### 3.2 加入群聊
```
GET http://localhost:1145/group/join?group_id=1708901234567
```

### 3.3 查看群聊列表
```
GET http://localhost:1145/group/list
```

### 3.4 查看群聊信息
```
GET http://localhost:1145/group/info?group_id=1708901234567
```

## 4. 使用群聊功能

### 4.1 发送群聊消息
在Talk页面 (`http://localhost:1145/talk`)：
- **用户输入框**：填写 `#群聊ID`（如 `#1708901234567`）
- **消息输入框**：填写要发送的内容
- 点击发送

### 4.2 读取群聊消息
同样在Talk页面：
- 用户输入框填写 `#群聊ID`
- 点击刷新或发送消息后自动刷新

## 5. 文件变化说明

### 5.1 新增文件
- `messages/groups.json` - 群聊数据存储

### 5.2 修改文件
1. `talk.py` - 添加群聊管理函数：
   - `create_group()` - 创建群聊
   - `join_group()` - 加入群聊  
   - `list_groups()` - 获取群聊列表
   - `group_info()` - 获取群聊信息
   - 修改了`read_message()`和`send_msg()`支持群聊

2. `Server.pyw` - 添加路由：
   - `/group/create`
   - `/group/join`
   - `/group/list`
   - `/group/info`

## 6. 技术细节

### 6.1 群聊标识
- 群聊ID以`#`开头
- 私聊：填写用户名
- 公共聊天：留空
- 群聊：`#群聊ID`

### 6.2 数据存储
- 群聊元数据：`messages/groups.json`
- 群聊消息：`msg_group_群聊ID.json`
- 消息格式与私聊相同

### 6.3 权限控制
- 只有群聊成员才能发送/读取消息
- 创建者自动成为成员
- 需要先加入群聊才能使用

## 7. 测试示例

```bash
# 1. 创建群聊
curl "http://localhost:1145/group/create?name=技术讨论群"

# 2. 加入群聊（假设返回group_id=123456）
curl "http://localhost:1145/group/join?group_id=123456"

# 3. 发送群聊消息（通过Talk页面更方便）
# 在浏览器访问：http://localhost:1145/talk
# 用户输入框填写：#123456
# 消息输入框填写：大家好！
```

## 8. 注意事项

1. **用户认证**：需要先登录/设置用户名
2. **群聊ID**：创建时自动生成，需要记录下来
3. **密钥**：群聊使用默认密钥，与私聊相同
4. **网络环境**：确保服务器可访问
